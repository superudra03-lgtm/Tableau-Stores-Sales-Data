VR Store Sales Data Analysis

This project analyzes retail sales data from multiple store locations to understand purchasing patterns, sales trends, and overall performance. The goal is to identify key insights that can help improve sales strategies and business decision-making.

The dataset includes information such as store location, store name, transaction date and time, month, day, hour, product SKU, quantity sold, and product price. By examining these variables, the analysis uncovers patterns in customer purchasing behavior across different times and locations.

Key insights from the data show how sales vary by time of day, day of the week, and store location. Certain hours and days tend to generate higher sales, indicating peak shopping periods. Store performance can also differ based on location, helping identify which stores are driving the most revenue. Additionally, product-level data such as SKU and quantity sold allows for identifying best-selling items and pricing trends.

The project was completed using Microsoft Excel for data cleaning, organization, and calculations, and Tableau for creating visual dashboards. These visualizations make it easier to analyze trends, compare store performance, and highlight important patterns in the data.

The purpose of this project is to provide actionable insights that can be used to optimize inventory management, improve staffing during peak hours, and enhance overall sales performance. By understanding when and where sales occur most frequently, businesses can make more informed decisions to increase efficiency and profitability.
